Introduction

This project demonstrates how to use an STM32 microcontroller's built-in timer to control an LED blinking pattern. Unlike traditional approaches that rely on delay functions, this method uses hardware timers for precise timing control, allowing the microcontroller to perform other tasks simultaneously.

The STM32 microcontroller family from STMicroelectronics offers powerful timers that can be configured for various timing and PWM (Pulse Width Modulation) applications. In this project, we'll explore basic timer configuration to toggle an LED at precise intervals.

What is Timer-Based LED Blinking?

Timer-based LED blinking is a method where a microcontroller's built-in timer peripheral is used to control the timing of LED state changes (on/off) instead of using software-based delay loops. The timer operates independently of the CPU, generating interrupts at precise intervals that trigger the LED to toggle state.

Advantages of Timer-Based Approach

Efficiency

Unlike delay-based methods, timer interrupts don't block the CPU, allowing it to perform other tasks.

Precision

Hardware timers provide much more accurate timing than software loops.

Flexibility

Easy to change timing parameters without restructuring code.

Scalability

Can control multiple LEDs with different timing patterns simultaneously.

Low Power

Allows CPU to enter low-power modes while timer handles timing.

Hardware Setup

  1. Connect the anode (long lead) of the LED to a GPIO pin (e.g., PA5) through a current-limiting resistor.
  2. Connect the cathode (short lead) of the LED to ground.
  3. Ensure your STM32 board is properly powered (USB or external power).
  4. The GPIO pin used should be configured as output push-pull.

STM32 Timer-Based LED Blinking Code

Here's the complete code for timer-based LED blinking using an STM32 microcontroller:

Main Code (main.c)


#include "main.h"

void SystemClock_Config(void);
static void MX_GPIO_Init(void);

int main(void)
{
    HAL_Init();
    SystemClock_Config();
    MX_GPIO_Init();

    while (1)
    {
        // LED ON
        HAL_GPIO_WritePin(GPIOA, GPIO_PIN_8, GPIO_PIN_SET);
        HAL_Delay(100);
        // LED OFF
        HAL_GPIO_WritePin(GPIOA, GPIO_PIN_8, GPIO_PIN_RESET);
        HAL_Delay(100);
    }
}

How the Code Works

  1. Initialization: The HAL library is initialized, the system clock is configured, and GPIO pins are initialized.
  2. Main Loop: The code enters an infinite loop where it turns the LED on and off with a delay of 100 milliseconds between each state change.
  3. LED Control: The function HAL_GPIO_WritePin() is used to set the state of the GPIO pin connected to the LED. GPIO_PIN_SET turns the LED on, while GPIO_PIN_RESET turns it off.
  4. Delay: The HAL_Delay(100) function creates a delay of 100 milliseconds, allowing the LED to remain in its current state for that duration.

Calculation Explanation

The LED blinks on and off every 100 milliseconds, resulting in a frequency of:

  • Frequency = 1 / (2 * 0.1 seconds) = 5 Hz
  • This means the LED will blink 5 times per second.
